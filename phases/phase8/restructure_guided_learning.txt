narrator_text (spoken aloud via TTS):
- Write as if a smart friend is explaining at a coffee shop
- Short sentences: 8-15 words each
- Use "you" and "your" â€” talk TO the learner directly

- ABSOLUTELY NO ANSI escape codes (e.g., [37m).

- Can use **bold** for key terms being introduced
- Can use bullet points for lists

- Must contain the KEY TERM the learner should remember
- ABSOLUTELY NO ANSI escape codes (e.g., [37m).

QUESTION DESIGN RULES (ADHD-OPTIMIZED)
â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•

Each Check Question MUST be answerable using ONLY the information presented in the learn segments of the CURRENT level or PRIOR levels in the same section. Do not assume prior knowledge of specific MCAT facts unless they were already taught in this section.

USE THESE 7 QUESTION TYPES. Vary them â€” never use the same type 3 times in a row:

1. "snap_judgment" â€” True or False, gut-check speed
   Schema: options: ["True", "False"], correct_index: 0 or 1
   Example: "True or false: DNA polymerase builds 3' to 5'."

2. "fill_the_gap" â€” One blank to complete
   Schema: options: ["choice A", "choice B", "choice C", "choice D"], correct_index: N
   Example: "Helicase _____ the double helix."

3. "pick_one" â€” 4-option multiple choice
   Schema: options: ["A", "B", "C", "D"], correct_index: N
   Example: "Which enzyme seals Okazaki fragments?"

4. "what_breaks" â€” Predict the consequence of a failure/inhibition
   Schema: options: ["A", "B", "C", "D"], correct_index: N
   Example: "Topoisomerase is blocked. What happens at the fork?"

5. "put_in_order" â€” Arrange 3-5 steps in correct sequence
   Schema: options: ["Step A", "Step B", "Step C", "Step D"], correct_index: N (index of correct ordering)
   Example: "Order: Seal â†’ Prime â†’ Unwind â†’ Extend"

6. "match_up" â€” Connect items from two parallel lists
   Schema: options: ["Item 1", "Item 2", "Item 3"], matches: ["Match 1", "Match 2", "Match 3"]
   *** CRITICAL: match_up MUST use SEPARATE "options" and "matches" arrays ***
   *** DO NOT put arrows in the options. DO NOT use correct_index for match_up. ***
   Example:
     "options": ["Helicase", "Ligase", "Primase"],
     "matches": ["Unwinds DNA", "Seals fragments", "Adds RNA primer"]

7. "spot_the_lie" â€” 3 true statements + 1 false. Find the false one.
   Schema: options: ["Statement A", "Statement B (false)", "Statement C", "Statement D"], correct_index: 1
   Example: "Which one is WRONG about the leading strand?"

QUESTION WORDING RULES:
- Start with an ACTION VERB or SCENARIO. Never "Which of the following..."
- Max 2 sentences for question stem
- For pick_one: 1 correct, 1 close distractor, 2 clearly wrong but plausible
- NEVER double negatives ("Which is NOT incorrect")
- NEVER "all of the above" or "none of the above"
- Use present tense: "What happens" not "What would happen"
- Frame as puzzles and challenges, not quiz questions

FEEDBACK FOR EVERY QUESTION:
- correct_response: 1 sentence celebration + WHY it's right. Never just "Correct!"
  Use LYRA's warm coaching tone: "Exactly! Helicase goes first because..." / "Nailed it! The fork needs..."
- wrong_response: Warm redirect + the actual answer. Frame as Grimble's corruption, not player failure.
  Examples: "Grimble twisted that one â€” here's the real answer..." / "Close â€” that's the corrupted version. The truth is..." / "Not this one â€” Grimble's fog is strong here. Think about..."
- wrong_explanations: For EACH wrong option, 1 sentence explaining the specific misconception
  (For match_up: provide a single wrong_response explaining what was mismatched)
